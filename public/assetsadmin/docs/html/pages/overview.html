Misc Pages

The extra pages in Ace's demo such as profile and login, are using common CSS classes and components and have no new elements, except a few additional CSS classes.

Profile

Profile page has a .sticky-nav-md element that becomes sticky on small screens (md and below):

html 
<div class="sticky-nav-md">
 <ul class="nav nav-tabs ...">
   ...
 </ul>
</div>

Using sticky-change event, when it becomes sticky, is-stuck event is added to it, to change the styling of .nav-tabs a little bit.

For more info please see:

  • Sticky Nav/Tabs
  • application/views/default/pages/partials/page-profile/@page-style.css
  • application/views/default/pages/partials/page-profile/@page-script.js

Login

As you can see in
application/views/default/layouts/login.hbs and application/views/default/pages/page-login.hbs
the default login page layout is like this:

html 
<div class="body-container">
  <div class="main-container container">
    <div role="main" class="main-content minh-100 justify-content-center">
      <div class="p-2 p-md-4">
        
        <div class="row">

          <div class="d-none d-lg-flex col-lg-5">
            <!-- this is the optional carousel and welcome message -->
            {{> page/intro}}
          </div>

          <div class="col-12 col-lg-7 py-lg-5 px-0">
            <div class="tab-content tab-sliding border-0 p-0" data-swipe="right">
          
              <div class="mh-100 tab-pane active show px-3 px-lg-0 pb-3" id="id-tab-login">
                <!-- login section -->
                {{> page/login}}
              </div>


              <div class="mh-100 tab-pane px-3 px-lg-0 pb-3" id="id-tab-signup" data-swipe-prev="#id-tab-login">
                <!-- signup section -->
                {{> page/signup}}
              </div>


              <div class="mh-100 tab-pane px-3 px-lg-0 pb-3" id="id-tab-forgot" data-swipe-prev="#id-tab-login">
                <!-- password reset section -->
                {{> page/forgot}}
              </div>

            </div><!-- .tab-content -->
          </div>

        </div>

      </div>
    </div><!-- /main -->
  </div><!-- /.main-container -->
</div><!-- /.body-container -->

You can see each section inside application/views/default/pages/partials/page-login folder.

For the second login style (with no intro & welcome message), you should remove the intro .col and make the following changes:

html 
<div class="row">
   <div class="col-12 col-lg-5 mx-auto py-lg-5 px-0">
        <div class="tab-content tab-sliding border-0 p-0" data-swipe="right">
            ...
        </div>
   </div>
</div>

For the third login style (fullscreen), you should make the following changes:

  • Remove container from main-container
  • Remove justify-content-center minh-100 from .main-content and add padding classes
  • Remove padding classes from the child and add d-flex, flex-column, etc
  • Add col-lg-6 (or other size) and mx-auto d-flex align-items-center justify-content-center to the main .col containing tab panes
html 
<div class="body-container">
  <div class="main-container bgc-white">
    <div role="main" class="main-content px-4 px-lg-0">
      <div class="d-flex flex-column flex-lg-row flex-grow-1 my-3 m-lg-0">
        
        <div class="row">

          <div class="d-none d-lg-flex col-lg-4">
            {{> page/intro}}
          </div>

          <div class="col-lg-6 mx-auto d-flex align-items-center justify-content-center py-lg-5 px-0">
            <div class="tab-content tab-sliding border-0 p-0" data-swipe="right">
                ...
            </div>
          </div>

        </div>

      </div>
    </div><!-- /main -->
  </div><!-- /.main-container -->
</div><!-- /.body-container -->

For more info about data-swipe="right" and data-swipe-prev="#id-tab-login" attributes, please see: Sliding Tabs

Also please see application/views/default/pages/partials/page-login/@page-style.css for additional styles used in login page.

Landing Page

Landing pages have no extra elements or special layout except a few CSS classes that you can see in:
application/views/default/pages/partials/landing-page-1/@page-style.css

Landing Page 1

An extra .navbar-compact class is defined, to make navbar compact and add a shadow to .navbar-inner.

The classname is added to .navbar as we scroll down and the optional #scroll-down element goes out of view (using IntersectionObserver).
And is removed when we scroll back up and the optional #scroll-up element reaches top of window.

See application/views/default/pages/partials/landing-page-1/@page-script.js for more info.

#scroll-down is an optional element and has only the top: 50vh property.
You can change it to a smaller value, so that navbar becomes fixed sooner or change it to a larger value so that navbar becomes fixed later.

Landing Page 2

There are some extra CSS rules to resize/position background images for header and footer according to screen size.
See application/views/default/pages/partials/landing-page-2/@page-style.css
