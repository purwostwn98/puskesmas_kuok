CSS Files

Dist files

Inside dist/css folder you can find CSS files used in demo.
However I don't recommend using them, because they contain custom CSS for all plugins, multiple sidebar features that you might not need and the default sidebar and navbar themes, which you might not use.

So I recommend building a custom CSS as described below and including your chosen theme as described in Choosing Default Themes

SASS and custom build

Ace's sass files are placed inside application/src/css and the main file is ace.scss .
In order to make changes and re-compile it you can start with modifying the following files:

  • _variables.scss (general variables such as colors, etc)
  • _variables-more.scss (more variables for components such as tabs, etc)
  • _features.scss (enable/disable features that you don't need, in order to reduce file size)
  • _sidebar-themes-variables.scss sidebar values and theme colors
  • _navbar-themes-variables.scss navbar values and theme colors
  • _pluging-*.scss files start with their variables

To make your CSS compatible with older browsers such as IE10 and Android 4.4, make sure you run npm run css-all or npm run css-prefix

Feature selection

As mentioned above to customize CSS output and remove unwanted features, you can modify _features.scss file.

For starters, you should defininely set only one of the $expandable , $hoverable , $hideable values to true.
Because you don't need three different minimized(collapsed) sidebar modes in your app. I enabled them all just for demo.

Or if you don't need Spaced Sidebars you should set $sidebar-spaced to be false .

Demo pages' CSS files

Some of Ace's demo pages have extra styling which I prefered not to include in the main ace.css because they are only used in that particular page.
For example the Error 404 page has some custom CSS rules for the animation, located at:
application/views/default/pages/partials/page-error/@page-style.css

Plugin CSS files

Most of Ace's demo pages use one or more plugins. For example for tables, date/time pickers, editors, etc and they have CSS files that you should include before ace.css

If so, the list of included stylesheets for each page can be found here:
application/views/default/pages/partials/PAGENAME/@vendor-stylesheets.hbs

As an example for Form Elements page you can see that
application/views/default/pages/partials/form-elements/@vendor-stylesheets.hbs
contains:

hbs 
{{{include-css files="nouislider + css/ion.rangeSlider.min.css@ion-rangeslider"}}}

which means the main CSS file of noUiSlider and css/ion.rangeSlider.min.css file from Ion.RangeSlider plugins are used in Form Elements page.

Note that include-css and include-js are custom Handlebars helpers used to specify and include requirements of each demo page during development of Ace.
You don't need to and should not use them.
