Ace Admin Template

First of all I would like to thank you for using Ace admin template v2.* .

Back in 2016 after releasing v1.4, I wasn't going to update Ace anymore because of personal circumstances.
Fortunately a while ago I got able to start working on and updating it again.

Please note that this documentation is brief and may be lacking in some parts. If you think more explanations and examples are needed, just let me know.

You should be familiar with Bootstrap 4 and jQuery to be able to use Ace and this documentation

Don't hesitate to contact me at AceAppThemes@gmail.com if you need support, have a comment or want to report a bug

You can also follow me on Twitter for updated news or for questions and comments

Features

Current release comes with the following features:

  • Responsive design based on Bootstrap 4
  • Four dashboard examples (3 layouts)
  • Multiple themes for sidebar and navbar and different customization options for each
  • Modular SASS and JS allowing to customize and include only the components you want
  • RTL support
  • Custom elements such as checkbox, switches, file input, sliding tabs, etc
  • Swiping gestures for sidebar and sliding tabs in mobile devices
  • Compatible with most browsers including IE10+, Android 4.4+, iOS 9+
  • And some more

Installation and Demo

Basic node_modules JS and CSS files to view the demo are included, but I recommend installing the latest updates.

After extracting Ace's zip file to a folder, you can install the required components and libraries such as Bootstrap, jQuery, etc by running the following command:

bash 
npm install
# You can also update packages to their latest version by running `npm update`

This will install components inside node_modules folder.

If you receive an error like this:

npm ERR! code ENOENT
npm ERR! syscall spawn git
...

It's because some third party modules depend on other modules that are installed directly from Github (and not from npm registry).
So you should install Git as well.

You may also consider bundling your app's required assets using a bundler such as webpack!

Next, to view the demo you can either open index.html in your browser or run the Node.js app by executing one of the following commands:

node index.js

Or use a different port:

PORT=6000 node index.js

and then browse the following address: http://localhost:8081/ (or the port your specified)

The index.js file along with the utils folder are just for running Ace's demo app and you don't need them
