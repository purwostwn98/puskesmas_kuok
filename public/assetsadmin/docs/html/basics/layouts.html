Layouts

Default

Default layout is used in most pages and has the following structure as you can view in application/views/default/layouts/main.hbs
In this layout navbar comes before .main-container

html 
<html>
  <head>
    <!-- title, meta tags, stylesheets, etc -->
  </head>

  <body>
   <div class="body-container">
      <div id="navbar" class="navbar navbar-default">
        <!-- navbar content -->
      </div><!-- #navbar -->

      <div class="main-container">
        
         <div id="sidebar" class="sidebar sidebar-default expandable">
           <!-- sidebar content -->
         </div><!-- #sidebar -->

         <div class="main-content" role="main">
          <div class="page-content">
           <!-- page's content -->
          </div><!-- .page-content -->

          <div id="footer" class="footer">
            <!-- footer content -->
          </div><!-- #footer -->
         </div><!-- main -->
      
      </div><!-- .main-container -->

      <!-- add required scripts, etc -->

   </div><!-- .body-container -->
  </body>
</html>

Head

As you can see in application/views/default/layouts/partials/_head.hbs, it contains title tag, meta tags (viewport, charset, etc), list of stylesheets used in all pages and in a specific page, fonts, and ace.css:

html 
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, height=device-height, initial-scale=1">

<title>Page Title</title>

<link href="/path/to/bootstrap.css" rel="stylesheet" />
<link href="/path/to/fontawesome or other icon set.css" rel="stylesheet" />
<link href="/path/to/my-selected-font.css" rel="stylesheet" />

<link href="/path/to/plugin1.css" rel="stylesheet" />
<link href="/path/to/plugin2.css" rel="stylesheet" />

<link href="/path/to/ace.css" rel="stylesheet" />

Tail

As you can see in application/views/default/layouts/partials/_tail.hbs, it contains required scripts:

html 
 <!-- include common vendor scripts used in demo pages -->
<script type="text/javascript" src="/path/to/jquery.js"></script>
<script type="text/javascript" src="/path/to/popper.js"></script>
<script type="text/javascript" src="/path/to/boostrap.js"></script>

<!-- include vendor scripts used in this page -->
<script type="text/javascript" src="/path/to/plugin1.js"></script>
<script type="text/javascript" src="/path/to/plugin2.js"></script>

<!-- include Ace script -->
<script type="text/javascript" src="/path/to/ace.js"></script>

<!-- include app script -->
<script type="text/javascript" src="/path/to/app.js"></script>

Dashboard 2 Layout

The layout in Dashboard 2 is a little different. See application/views/default/layouts/dashboard2.hbs .
Here, navbar is inside .main-content so sidebar start from top of the page.
And the .content-bg1 class applied to .main-container adds a background color to the the content area. See Content Area

html 
<html>
  <head>
    <!-- title, meta tags, stylesheets, etc -->
  </head>

  <body>
   <div class="body-container">

      <div class="main-container content-bg1">
        
         <div id="sidebar" class="sidebar sidebar-white expandable">
          {{> layout/layout-2/sidebar}}
         </div>

         <div role="main" class="main-content">
          {{> layout/layout-2/navbar}}

          <div class="page-content container">
           {{{body}}} <!-- page's content -->
          </div> <!-- .page-content -->

          {{> layout/layout-2/footer}}
         </div> <!-- main -->
      
      </div> <!-- .main-container -->

   </div> <!-- .body-container -->
  </body>
</html>

Dashboard 3 Layout

The layout in Dashboard 3 is the same as the main layout. There are only some changes to initial navbar and sidebar themes and boxed layout.

Login Layout

Login page has no sidebar or navbar and it has 3 options:

  • The default one when you open login page
  • The compact one when you click Remove this section
  • Fullpage (fullscreen) view when you click Become full size

Please refer to Login Page for more details.

Horizontal Menu Layout

Horizontal menu is not different. You sould add required class names to sidebar. See here
It can be inserted right after .navbar or inside content area as described below in Two Menus Layout.

Two Menus Layout

You can have two (or more) menus. There are examples in Ace's demo.

Example 1

Like in Two Menus 1 demo page, main menu is vertical (on left or right) and 2nd menu is horizontal inside content area:

html 
<div class="main-container">
 <div class="sidebar expandable sidebar-fixed" id="sidebar1" data-dismiss="true" data-backdrop="true">
   <!-- vertical sidebar -->
 </div>

 <div class="main-content">
    <div class="navbar">
      <!-- optional navbar -->
    </div>

    <div class="sidebar sidebar-h sidebar-hover collapsed-h" id="sidebar2">
      <!-- horizontal sidebar -->
    </div>

    ...
 </div>
</div>

Example 2

Like in Two Menus 2 demo page, main menu is horizontal and 2nd menu is vertical inside main container:

html 
<div class="navbar">
 ...
</div>

<div class="sidebar sidebar-fixed sidebar-hover sidebar-h" id="sidebar1">
 ...
</div>

<div class="main-container" id="sidebar2">
  <div class="sidebar expandable collapsed">
   ...
  </div>
</div>
