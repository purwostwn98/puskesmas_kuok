Themes

Choosing default themes

Ace has a few coloring options for sidebar and navbar.

If you want to use a theme in your app other than the default one, it's recommended to compile ace.css using your chosen theme(s).
You should not use ace-themes.css because it contains all the themes and is a large file.

For that purpose, open application/css/src/ace-default-themes.scss and add or remove the theme(s) you want.

ðŸ‘‰ Note that for Dark Colored Sidebars you should include and compile both sidebar-color and the sidebar you want.
For example:

scss 
$my-navbar-themes: (
    "blue": $navbar-blue
);
$my-sidebar-themes: (
    "white": $sidebar-white
);
// OR if you want a dark sidebar like sidebar-purple
$my-sidebar-themes: (
    "color": $sidebar-color, // sidebar-color should be included as well
    "purple": $sidebar-purple
);

Then rebuild ace.css

And you can change navbar and sidebar like this:

html 
<div id="navbar" class="navbar navbar-blue navbar-fixed">
 <!-- navbar content -->
</div>
html 
<div id="sidebar" class="sidebar sidebar-color sidebar-blue sidebar-fixed">
 <!-- sidebar content -->
</div>

Customizing And Editing

If you want to change the colors and other values of a theme, you should edit application/css/src/_sidebar-themes-variables.scss

You can customize each theme to change their colors or disable/enable features.
Many themes are extended from another theme. For example sidebar-white is extended from sidebar-light.

ðŸ‘‰ Some of the more important options are:

item-border-color : adds a border between sidebar items. Can be used with item-border-margin to specify distance from sides

border-color and shadow : some sidebars have a border on right (for example white sidebars), some have a shadow (for example the default sidebar), some have none (for example dark sidebars) .

With a shadow you can have the .active element cover the shadow by setting shadow-padding to 1px (or more) and result is like this:

But it's not possible with a border. So if you want a border and not a shadow:

Then you should modify your theme values (in _sidebar-themes-variables.scss) like this:

scss 
$sidebar-default: map-merge($sidebar-light, (
    'background-color': #f7f7f7,
    'border-color': #dbdfe2,
    'shadow': null,
    ....
));

However when active item covers sidebar's shadow in collapsed/minimized mode, it sometimes doens't look good. And you can fix that by adding the following theme variable:

scss 
$sidebar-default: map-merge($sidebar-light, (
    'background-color': #f7f7f7,
    'border-color': null,
    'shadow': (inset -1px 0 0 0 #dbdfe2),
    'shadow-padding': 1px,
    'shadow-padding-collapsed-disabled': true,
	// OR
	'shadow-padding-collapsed-active-on-right': true,//will show the active highligh border on right, when sidebar is collapsed and hovered
    ....
));


or when active item highlight is on right: (Explained in Sidebar Nav Options)

If you want to add a 3D shadow to sidebar, you should add it to its .sidebar-inner child:

html 
<div id="sidebar" class="sidebar sidebar-white">
  <div class="sidebar-inner shadow-sm">
      ...
  </div>
</div>

submenu-bullets disables or changes submenu bullets for a theme:


* null disables it * 1 only shows it on hover * 2 only shows when item is .active * 3 shows it both on hover and when .active * set submenu-bullet-style to caret to change it from circle to triangle

scss 
$sidebar-purple: (
    ...
    'submenu-bullets': 1,
    'submenu-bullet-style': caret  
    ...
);

link-hover-highlight-border . Sidebar link (A.nav-link) border when hovered. null disables it:

link-active-highlight-border . Sidebar link (A.nav-link) border when .active:

item-active-highlight-border . Sidebar item (LI.nav-item) border when .active:

submenu-tree-border . If true submenu items will have tree like borders. Similar to previous image.

If you want to change the colors and other values of a theme, you should edit application/css/src/_navbar_-themes-variables.scss

Then you should rebuild ace.css using your chosen theme and use it like this:

html 
<div class="navbar navbar-white"> ... </div>

<div class="navbar navbar-lightblue"> ... </div>

<div class="navbar navbar-purple"> ... </div>

In Ace's demo, when you change navbar theme, some of the elements are also changed.
These changes are not part of the theme. Instead their relevant class names are changed.

As an example:

html 
<a href="#" class="dropdown-toggle nav-link pl-lg-3 pr-lg-4">
    ...
</a>
<!-- ↓ BECOMES ↓ -->
<a href="#" class="dropdown-toggle btn btn-warning px-lg-3 d-none d-lg-flex">
    ...
</a>

Here .nav-link class is replaced by .btn.btn-warning.

Also in the second theme (lightblue), the .btn is displayed only on large (desktop) devices by applying d-none d-lg-flex .
In small devices a sibling is inserted and displayed instead to match the look and feel of the large screen:

html 
<!-- Show the "orange" button in large screens -->
<a href="#" class="dropdown-toggle btn btn-warning px-lg-3 d-none d-lg-flex">
    ...
</a>
<!-- 
    And in small devices show a normal ".nav-link"
    and place an "orange" icon inside it   
-->
<a href="#" class="dropdown-toggle nav-link d-lg-none">    
    <i class="fa fa-bell icon-animated-bell 
        btn-warning radius-round w-4 h-4 text-center pt-2 text-110">
    </i>
    ...
</a>

Body

There are some background color/image options for body element.
These classes also effect the border(shadow) around .body-container.container in boxed layout
The following are available:

  • body-img1
  • body-img2
  • body-darkblue , body-darkblue2 , body-darkblu3
  • body-lightblue , body-lightblue , body-lightblue3 , body-lightblue4
  • body-darkslategrey
  • body-darkplum , body-lightplum
  • body-darkgreen , body-lightgreen
html 
<body class="body-img1">
...
</body>

You can disable these by setting $body-bg-themes to false in _features.scss .
You can also change the values in application/css/src/ace-themes . Search for $body-bg-themes in it

Content Area

There is only one extra background color for .main-container but you can use classes like bgc-default-l3, etc to change content area's background:

html 
<div class="main-container content-bg1">
  ...
</div>
<!-- or -->
<div class="main-container bgc-secondary-l4">
  ...
</div>

If you want to use a non-white background color for content area (like in Dashboard 2 example) and you have sidebar arrows that point to .active item:

you should make sure to add the following rule in your CSS so that the arrow color matches content area background:

css 
.sidebar .nav.has-active-arrow .nav-item.active > .nav-link::after {
    border-right-color: #eee; /* put the same color as content area's background color */
}
